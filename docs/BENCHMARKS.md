# ANDRII Benchmarks

Generated by `cargo run -p andrii-core --example benchmark -- --full` (debug build — release
is blocked by a missing `dlltool` on the `x86_64-pc-windows-gnu` toolchain; see [#toolchain]
note below).

> **Numbers below are from a debug build and are therefore slower than production.**
> Timings will drop substantially in a release build, but the relationships
> (media stays ~input, text shrinks) are invariant.

Datasets: **small-mixed** (compressible text + small media), **many-small** (1000 JSON), **large-media** (incompressible blobs ≈ already-compressed media), **large-bin** (one 1024 MiB random file, streaming-path stress).

`saved%` is computed against the on-disk archive size (includes headers, per-chunk AEAD tags and footer), so tiny inputs can show slightly negative savings.

## Notes

- **Media & binary stay ≈input across all modes** — incompressible data is detected and stored raw, so Maximum no longer wastes minutes for ~0% gain.
- **Text/JSON shrink**, and Maximum < Balanced < Fast on output size for those datasets.
- The large-bin run completes with bounded memory (≈ one 1 MiB chunk) thanks to v2 chunked streaming.
